# 🧑‍💻 신규 프로젝트 멀티 에이전트 가이드

이 저장소는 신규 프로젝트 개발을 빠르게 진행하기 위해 메인 오케스트레이터 + 전문 서브 에이전트 구조를 사용합니다.

## 기본 원칙

1. 사용자에게 보이는 모든 응답과 되물음은 한국어로만 작성합니다.
2. 각 에이전트는 실행 시 첫 줄에 "... 에이전트 구동" 문구를 출력합니다.

## 에이전트 목록과 역할

### 1) 메인 오케스트레이터

- 파일: `.github/agents/main-orchestrator.agent.md`
- 역할:
  - 현재 개발 단계 판단
  - 적절한 서브 에이전트 선택 및 위임
  - 전체 진행 흐름 관리
- 구동 문구:
  - 메인 오케스트레이터 에이전트 구동

### 2) 요구사항정리 에이전트

- 파일: `.github/agents/requirements-planner.agent.md`
- 역할:
  - 사용자 요구사항 구체화
  - MVP/우선순위/수용 기준 정의
  - 리스크 및 가정 정리
- 구동 문구:
  - 요구사항정리 에이전트 구동

### 3) 기술스택 정리 에이전트

- 파일: `.github/agents/tech-stack-planner.agent.md`
- 역할:
  - 기술스택 선정
  - 런타임/프레임워크/패키지 매니저 버전 정책 수립
  - 호환성 점검 및 업데이트 정책 제시
- 구동 문구:
  - 기술스택 정리 에이전트 구동

### 4) 프로젝트 구조 정리 에이전트

- 파일: `.github/agents/project-architect.agent.md`
- 역할:
  - 폴더 구조, 모듈 경계, 의존 방향 정의
  - 공통 규칙(네이밍/설정/환경 전략) 제시
- 구동 문구:
  - 프로젝트 구조 정리 에이전트 구동

### 5) FE개발 에이전트

- 파일: `.github/agents/frontend-developer.agent.md`
- 역할:
  - UI/상태/라우팅/폼/접근성/반응형 구현
  - 프론트 품질 점검
- 구동 문구:
  - FE개발 에이전트 구동

### 6) BE개발 에이전트

- 파일: `.github/agents/backend-developer.agent.md`
- 역할:
  - API/도메인 로직/데이터 계층 구현
  - 서버 안정성과 운영 관점 점검
- 구동 문구:
  - BE개발 에이전트 구동

### 7) 코드 검증 에이전트

- 파일: `.github/agents/code-validation-refactor.agent.md`
- 역할:
  - FE/BE 산출물 오류 검증
  - 안전한 리팩토링 및 재검증
- 구동 문구:
  - 코드 검증 에이전트 구동

### 8) 보안검증 에이전트

- 파일: `.github/agents/security-auditor.agent.md`
- 역할:
  - 인증/인가/입력검증/비밀정보 노출 등 보안 리스크 점검
  - 심각도 기반 우선순위 제시
- 구동 문구:
  - 보안검증 에이전트 구동

## 권장 실행 순서

1. 요구사항정리
2. 기술스택 정리
3. 프로젝트 구조 정리
4. FE개발 / BE개발
5. 코드 검증
6. 보안검증

## 빠른 시작 예시 프롬프트

### 메인 오케스트레이터에게

- 신규 프로젝트를 시작할게. 요구사항 정리부터 어떤 순서로 진행할지 계획해줘.

### 요구사항정리 에이전트에게

- 오프라인 매장 재고관리 웹앱 MVP 요구사항을 정리해줘.

### 기술스택 정리 에이전트에게

- React + Node 기반으로 시작하려고 해. 권장 버전과 호환성, 버전 핀 정책을 제안해줘.

### 프로젝트 구조 정리 에이전트에게

- FE/BE 분리형 구조로 폴더 구조와 모듈 경계를 설계해줘.

### FE개발 에이전트에게

- 상품 목록/검색/필터 화면을 접근성과 반응형 기준으로 구현해줘.

### BE개발 에이전트에게

- JWT 인증 포함한 상품 CRUD API를 구현해줘.

### 코드 검증 에이전트에게

- 최근 FE/BE 변경분을 검증하고, 필요한 리팩토링을 적용해줘.

### 보안검증 에이전트에게

- 인증/인가/입력검증/비밀정보 노출 관점으로 보안 이슈만 점검해줘.

## 에이전트 파일 위치

- `.github/agents/`

이 저장소를 clone한 뒤 VS Code에서 열면 에이전트 설정을 바로 사용할 수 있습니다.
